# BMP388 -- Digital Pressure Sensor -- Technical Reference

**Manufacturer:** Bosch Sensortec
**Document:** BST-BMP388-DS001-07, Rev 1.7, November 2020
**Package:** 10-pin metal-lid LGA, 2.0 x 2.0 x 0.75 mm

## 1. Device Overview

The BMP388 is a digital barometric pressure and temperature sensor by Bosch Sensortec. It uses a piezo-resistive pressure sensing element and a mixed-signal ASIC that performs A/D conversion and provides factory-calibrated compensation data through a digital interface (I2C or SPI).

- **Pressure range:** 300 -- 1250 hPa
- **Temperature range (operational):** -40 to +85 C
- **Temperature range (full accuracy):** 0 to +65 C
- **Chip ID register value:** 0x50
- **Interfaces:** I2C (up to 3.4 MHz), SPI (3-wire and 4-wire, up to 10 MHz)
- **FIFO:** 512-byte buffer
- **Interrupts:** Data ready, FIFO watermark, FIFO full

## 3. Timing Requirements

### 3.1 Startup

- **Start-up time:** 2 ms (time to first communication after both VDD > 1.58V and VDDIO > 0.65V)
- After power-on reset, the sensor settles in **sleep mode**
- Completion of POR is indicated by `por_detected` bit (register 0x10, bit 0); cleared on read

### 3.2 Measurement Time

Conversion time formula (in microseconds):

```
T_conv = 234 + press_en * (392 + 2^osr_p * 2000) + temp_en * (313 + 2^osr_t * 2000)
```

| Oversampling          | Press OS | Temp OS | Typ (ms) | Max (ms) | Max Rate (Hz) |
| --------------------- | -------- | ------- | -------- | -------- | ------------- |
| Ultra low power       | x1       | x1      | 4.82     | 5.70     | 175           |
| Low power             | x2       | x1      | 6.84     | 7.96     | 126           |
| Standard resolution   | x4       | x1      | 10.88    | 12.48    | 80            |
| High resolution       | x8       | x1      | 18.69    | 21.53    | 46            |
| Ultra high resolution | x16      | x2      | 37.14    | 41.89    | 24            |
| Highest resolution    | x32      | x2      | 69.46    | 78.09    | 13            |

### 3.3 Maximum Sampling Rate

- **f_sampling base:** 200 Hz
- **Normal mode:** sampling period = prescaler / 200 Hz (prescaler set via `odr_sel` register)
- **Forced mode:** maximum frequency = 1 / T_conv

## 5. I2C Interface Details

### 5.1 I2C Addresses

The 7-bit address is `111011X`:

| SDO Pin | Address (7-bit) | Address (8-bit write) | Address (8-bit read) |
| ------- | --------------- | --------------------- | -------------------- |
| GND     | 0x76            | 0xEC                  | 0xED                 |
| VDDIO   | 0x77            | 0xEE                  | 0xEF                 |

The SDO pin must NOT be left floating -- the address will be undefined.

### 5.2 I2C Protocol

- Compatible with Philips I2C Specification Rev 6 (standard, fast, high speed modes)
- SCK frequency: 400 kHz (typ), max 3400 kHz
- SDA and SCL are NOT pure open-drain; they contain ESD protection diodes to VDDIO and GND
- Internal pull-up on CSB: 75-125 kOhm (typ 100 kOhm)
- Bus load capacitance on SDI and SCK: max 400 pF

### 5.3 I2C Watchdog

- Configurable via `IF_CONF` register (0x1A)
- `i2c_wdt_en` (bit 1): enable/disable watchdog timer
- `i2c_wdt_sel` (bit 2): short timeout (1.25 ms) or long timeout (40 ms)
- Backed by NVM

## 6. Power Modes

Controlled by `mode[1:0]` bits in `PWR_CTRL` register (0x1B, bits 5:4):

| mode[1:0] | Mode   | Description                                                                    |
| --------- | ------ | ------------------------------------------------------------------------------ |
| 00        | Sleep  | Default after POR. No measurements. All registers accessible. Minimum current. |
| 01 or 10  | Forced | Single measurement, then returns to sleep. Must re-trigger for each reading.   |
| 11        | Normal | Continuous cyclic measurements at rate set by ODR register.                    |

### 6.1 Mode Transitions

- Power-on always enters Sleep
- Sleep -> Forced: write mode=01 or mode=10
- Sleep -> Normal: write mode=11
- Normal -> Sleep: write mode=00
- Forced -> Sleep: automatic after measurement completes
- If a measurement is in progress, mode switching is delayed until it completes

## 7. Measurement Configuration

### 7.1 Enabling Sensors

In `PWR_CTRL` register (0x1B):

- Bit 0 (`press_en`): 0 = pressure disabled, 1 = pressure enabled
- Bit 1 (`temp_en`): 0 = temperature disabled, 1 = temperature enabled

### 7.2 Oversampling (register 0x1C "OSR")

**Pressure oversampling** -- bits [2:0] (`osr_p`):

| osr_p | Oversampling | Resolution        | Noise (Pa, IIR off) |
| ----- | ------------ | ----------------- | ------------------- |
| 000   | x1           | 16 bit / 2.64 Pa  | 6.6                 |
| 001   | x2           | 17 bit / 1.32 Pa  | 4.3                 |
| 010   | x4           | 18 bit / 0.66 Pa  | 3.2                 |
| 011   | x8           | 19 bit / 0.33 Pa  | 2.3                 |
| 100   | x16          | 20 bit / 0.17 Pa  | 1.6                 |
| 101   | x32          | 21 bit / 0.085 Pa | 1.2                 |

**Temperature oversampling** -- bits [5:3] (`osr_t`):

| osr_t | Oversampling | Resolution         |
| ----- | ------------ | ------------------ |
| 000   | x1           | 16 bit / 0.0050 C  |
| 001   | x2           | 17 bit / 0.0025 C  |
| 010   | x4           | 18 bit / 0.0012 C  |
| 011   | x8           | 19 bit / 0.0006 C  |
| 100   | x16          | 20 bit / 0.0003 C  |
| 101   | x32          | 21 bit / 0.00015 C |

**Recommendation:** Temperature oversampling above x2 does not significantly improve compensated pressure accuracy.

### 7.3 IIR Filter (register 0x1F "CONFIG")

Bits [3:1] (`iir_filter`):

| Value | Coefficient      |
| ----- | ---------------- |
| 000   | 0 (bypass / off) |
| 001   | 1                |
| 010   | 3                |
| 011   | 7                |
| 100   | 15               |
| 101   | 31               |
| 110   | 63               |
| 111   | 127              |

Filter formula: `data_filtered = (data_filtered_old * coeff + data_ADC) / (coeff + 1)`

Writing to the filter register resets the filter. The filter is also reset when `temp_en` or `press_en` changes from 1 to 0, or on sleep-to-normal transition.

Bits [4:0] (`odr_sel`), prescaler = 2^odr_sel, f_sampling = 200 Hz:

### 9.2 Coefficient Conversion (NVM raw -> floating point)

**Temperature coefficients:**

```
PAR_T1 = NVM_PAR_T1 / 2^-8       (i.e. NVM_PAR_T1 * 256.0)
PAR_T2 = NVM_PAR_T2 / 2^30
PAR_T3 = NVM_PAR_T3 / 2^48
```

**Pressure coefficients:**

```
PAR_P1  = (NVM_PAR_P1 - 2^14) / 2^20
PAR_P2  = (NVM_PAR_P2 - 2^14) / 2^29
PAR_P3  = NVM_PAR_P3  / 2^32
PAR_P4  = NVM_PAR_P4  / 2^37
PAR_P5  = NVM_PAR_P5  / 2^-3      (i.e. NVM_PAR_P5 * 8.0)
PAR_P6  = NVM_PAR_P6  / 2^6
PAR_P7  = NVM_PAR_P7  / 2^8
PAR_P8  = NVM_PAR_P8  / 2^15
PAR_P9  = NVM_PAR_P9  / 2^48
PAR_P10 = NVM_PAR_P10 / 2^48
PAR_P11 = NVM_PAR_P11 / 2^65
```

### 9.3 Temperature Compensation Algorithm

Input: `uncomp_temp` (unsigned 24-bit raw value from DATA_3/4/5)
Output: compensated temperature in degrees C (float)

```c
float partial_data1 = (float)(uncomp_temp - par_t1);
float partial_data2 = (float)(partial_data1 * par_t2);
// t_lin is stored for pressure compensation
t_lin = partial_data2 + (partial_data1 * partial_data1) * par_t3;
return t_lin;  // compensated temperature in C
```

### 9.4 Pressure Compensation Algorithm

Input: `uncomp_press` (unsigned 24-bit raw value from DATA_0/1/2), and `t_lin` from temperature compensation
Output: compensated pressure in Pa (float)

```c
// partial_out1
partial_data1 = par_p6 * t_lin;
partial_data2 = par_p7 * (t_lin * t_lin);
partial_data3 = par_p8 * (t_lin * t_lin * t_lin);
partial_out1  = par_p5 + partial_data1 + partial_data2 + partial_data3;

// partial_out2
partial_data1 = par_p2 * t_lin;
partial_data2 = par_p3 * (t_lin * t_lin);
partial_data3 = par_p4 * (t_lin * t_lin * t_lin);
partial_out2  = (float)uncomp_press *
                (par_p1 + partial_data1 + partial_data2 + partial_data3);

// partial_data4
partial_data1 = (float)uncomp_press * (float)uncomp_press;
partial_data2 = par_p9 + par_p10 * t_lin;
partial_data3 = partial_data1 * partial_data2;
partial_data4 = partial_data3 +
                ((float)uncomp_press * (float)uncomp_press * (float)uncomp_press) * par_p11;

comp_press = partial_out1 + partial_out2 + partial_data4;
return comp_press;  // compensated pressure in Pa
```

**IMPORTANT:** Temperature must always be compensated first, as `t_lin` is required for pressure compensation.

## 10. Data Readout

### 10.1 Burst Read

Use a burst read from register 0x04 through 0x09 (6 bytes) to read both pressure and temperature atomically. Data register shadowing ensures consistency only when a single burst read covers all data registers.

Pressure (unsigned 24-bit): `(DATA_2 << 16) | (DATA_1 << 8) | DATA_0`
Temperature (unsigned 24-bit): `(DATA_5 << 16) | (DATA_4 << 8) | DATA_3`

### 10.2 Data Ready Checking

Two methods:

1. **Polling:** Read STATUS register (0x03): `drdy_press` (bit 5), `drdy_temp` (bit 6). These bits are reset when any corresponding DATA register is read.
2. **Interrupt:** Enable `drdy_en` in INT_CTRL (0x19, bit 6). Read INT_STATUS (0x11, bit 3) to check/clear.

---

## 11. Initialization Sequence

A typical driver initialization for forced-mode weather monitoring:

1. **Wait for startup:** 2 ms after power supplies reach threshold
2. **Read CHIP_ID** (0x00): verify it returns **0x50**
3. **Soft reset:** write 0xB6 to CMD register (0x7E); wait 2 ms; read EVENT register (0x10) to confirm `por_detected`
4. **Read calibration data:** burst read 0x31-0x45 (21 bytes); parse into NVM*PAR_T1..NVM_PAR_P11; convert to floating-point PAR*\* coefficients
5. **Configure oversampling:** write OSR register (0x1C) -- e.g., 0x00 for x1/x1
6. **Configure IIR filter:** write CONFIG register (0x1F) -- e.g., 0x00 for off
7. **Configure ODR** (if using normal mode): write ODR register (0x1D)
8. **Configure interrupts** (optional): write INT_CTRL register (0x19)
9. **Enable sensors and set mode:** write PWR_CTRL register (0x1B)
   - For forced mode: `0x13` = press_en + temp_en + mode=forced (bits: 0001 0011)
   - For normal mode: `0x33` = press_en + temp_en + mode=normal (bits: 0011 0011)
10. **Wait for measurement** (forced mode): poll STATUS register for `drdy_press` and `drdy_temp`, or wait for the maximum conversion time
11. **Read data:** burst read 0x04-0x09
12. **Compensate:** run temperature compensation, then pressure compensation

For forced mode, repeat from step 9 for each new measurement.

---

## 12. FIFO

- 512-byte buffer; enabled via FIFO_CONFIG_1 (0x17)
- Each frame has a 1-byte header followed by data bytes
- Sensor frame (temp+press): 7 bytes (1 header + 3 temp + 3 press)
- Sensor frame (temp or press only): 4 bytes
- Frame header bit 7 is always 1 for sensor frames; bits [4:3] indicate time/temp/press enables
- Overflow: configurable to either overwrite oldest (streaming) or stop (fifo_stop_on_full)
- Flush: write 0xB0 to CMD register, or soft reset (0xB6)
- Read via repeated reads of FIFO_DATA register (0x14) -- address stays at 0x14 during burst
