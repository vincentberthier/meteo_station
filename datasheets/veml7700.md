# VEML7700 -- High Accuracy Ambient Light Sensor

**Manufacturer:** Vishay Semiconductors
**Datasheet Rev:** 1.8, 28-Nov-2024

## Device Overview

16-bit ambient light sensor in a 6.8 x 2.35 x 3.0 mm package. Integrates photodiode, amplifier, 16-bit ADC, and I2C interface. Two channels: ALS (photopic response) and WHITE (broader spectral sensitivity).

- **Range:** 0 to ~140,000 lx
- **Best resolution:** 0.0042 lx/count
- **Flicker rejection:** 100 Hz and 120 Hz
- **Shutdown current:** 0.5 uA typ

## I2C Protocol

- **Address:** 0x10 (7-bit, fixed, not configurable)
- All registers 16-bit, LSB first
- Pull-ups: > 1 kOhm, recommended 2.2-4.7 kOhm
- No interrupt pin -- polling required

## Register Map

| Cmd  | Name         | R/W | Description                                                 |
| ---- | ------------ | --- | ----------------------------------------------------------- |
| 0x00 | ALS_CONF_0   | R/W | Configuration (gain, integration time, interrupt, shutdown) |
| 0x01 | ALS_WH       | R/W | High threshold (16-bit)                                     |
| 0x02 | ALS_WL       | R/W | Low threshold (16-bit)                                      |
| 0x03 | Power Saving | R/W | PSM mode and enable                                         |
| 0x04 | ALS          | R   | Ambient light output (16-bit)                               |
| 0x05 | WHITE        | R   | White channel output (16-bit)                               |
| 0x06 | ALS_INT      | R   | Interrupt status (bit 15: low, bit 14: high)                |
| 0x07 | ID           | R   | Device ID (low byte = 0x81)                                 |

## Configuration Register (0x00)

| Bits  | Field      | Description                          |
| ----- | ---------- | ------------------------------------ |
| 12:11 | ALS_GAIN   | 00=x1, 01=x2, 10=x1/8, 11=x1/4       |
| 9:6   | ALS_IT     | Integration time (see below)         |
| 5:4   | ALS_PERS   | Persistence (00=1, 01=2, 10=4, 11=8) |
| 1     | ALS_INT_EN | 0=disable, 1=enable interrupt        |
| 0     | ALS_SD     | 0=power on, 1=shutdown               |

**Default: 0x0001** (shutdown mode).

### Integration Time (bits 9:6)

| Value | Time   |
| ----- | ------ |
| 1100  | 25 ms  |
| 1000  | 50 ms  |
| 0000  | 100 ms |
| 0001  | 200 ms |
| 0010  | 400 ms |
| 0011  | 800 ms |

### Power Saving Mode (register 0x03)

| Bits | Field  | Description                                |
| ---- | ------ | ------------------------------------------ |
| 2:1  | PSM    | 00=mode 1, 01=mode 2, 10=mode 3, 11=mode 4 |
| 0    | PSM_EN | 0=disable (continuous), 1=enable           |

## Lux Calculation

```
Lux = ALS_raw * resolution
```

### Resolution Table

| Gain | Integration Time | Resolution (lx/count) |
| ---- | ---------------- | --------------------- |
| x2   | 800 ms           | 0.0042                |
| x2   | 400 ms           | 0.0084                |
| x2   | 200 ms           | 0.0168                |
| x2   | 100 ms           | 0.0336                |
| x1   | 100 ms           | 0.0672                |
| x1/4 | 100 ms           | 0.2688                |
| x1/8 | 100 ms           | 0.5376                |
| x1/8 | 25 ms            | 2.1504                |

General formula: `resolution = 0.0042 * (800 / ALS_IT_ms) * (2 / gain_multiplier)`

### Nonlinearity Correction (above ~1000 lx)

```
lux_corrected = 6.0135e-13 * lux^4 - 9.3924e-09 * lux^3 + 8.1488e-05 * lux^2 + 1.0023 * lux
```

## Initialization and Reading

1. Write 0x00 with desired config (clear ALS_SD to power on)
2. (Optional) Configure PSM in register 0x03
3. Wait one integration period
4. Read register 0x04 for ALS data
5. Convert to lux using resolution table
6. Apply correction polynomial if > 1000 lx

### Verify communication: read register 0x07, low byte should be 0x81

## Driver Notes

- Powers up in shutdown mode -- must clear ALS_SD
- After changing gain/integration time, discard first reading
- Fixed I2C address 0x10 -- no configuration available
- No physical interrupt pin despite register support
