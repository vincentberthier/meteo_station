# RN4870/RN4871 BLE Module -- Technical Reference

**Manufacturer:** Microchip
**Sources:** DS50002489A (Data Sheet), DS50002466C (User Guide)

## Device Overview

Bluetooth 4.2 Low Energy modules based on Microchip IS1870 SoC. Host MCU controls them via ASCII commands over UART.

### RN4870 vs RN4871

| Feature      | RN4870   | RN4871    |
| ------------ | -------- | --------- |
| Size         | 12x22 mm | 9x11.5 mm |
| Pins         | 33       | 16        |
| SPI          | Yes      | No        |
| PWM          | Up to 3  | No        |
| ADC channels | 6        | 4         |
| I2C          | Yes      | Yes       |

## UART Protocol

### Default Settings

| Parameter    | Value    |
| ------------ | -------- |
| Baud Rate    | 115200   |
| Data Bits    | 8        |
| Parity       | None     |
| Stop Bits    | 1        |
| Flow Control | Disabled |

### Baud Rates (SB command)

| Setting | Rate                 |
| ------- | -------------------- |
| 00      | 921600               |
| 01      | 460800               |
| 02      | 230400               |
| **03**  | **115200 (default)** |
| 04      | 57600                |
| 05      | 38400                |
| 07      | 19200                |
| 09      | 9600                 |

### Command Protocol

- Commands end with `\r` (0x0D), no line feed
- 1-3 case-insensitive ASCII chars + optional params separated by commas
- Wait for response before sending next command
- Success: `AOK\r\n`, failure: `ERR\r\n`
- Prompt: `CMD> ` (unless suppressed with SR bit 0x4000)

---

## Operating Modes

### Data Mode (default)

UART data passes through to BLE peer via Transparent UART service.

### Command Mode

- **Enter:** `$$$` (100 ms delay before first `$`). Response: `CMD>`
- **Exit:** `---\r`. Response: `END`

### Fast Data Mode (SR bit 0x2000)

`$$$` disabled. Use PIO pin configured as "UART Mode Switch" (function 0x0C).

## Command Reference

### Set Commands (stored in NVM, need reboot)

| Command                               | Description                                      |
| ------------------------------------- | ------------------------------------------------ |
| `SN,<text>`                           | Device name (up to 20 chars)                     |
| `SS,<hex8>`                           | Default services bitmap                          |
| `SR,<hex16>`                          | Feature bitmap                                   |
| `SA,<0-5>`                            | Authentication I/O capability                    |
| `SB,<hex8>`                           | UART baud rate                                   |
| `SP,<pin>`                            | Security PIN (default: 000000)                   |
| `SF,1`                                | Factory reset (keep services). Immediate reboot. |
| `SF,2`                                | Full factory reset. Immediate reboot.            |
| `SGA,<0-5>`                           | TX power during advertising                      |
| `SGC,<0-5>`                           | TX power during connection                       |
| `SO,<0,1>`                            | Low-power mode (32 kHz clock)                    |
| `ST,<min>,<max>,<lat>,<to>`           | Connection parameters                            |
| `STA,<fast_int>,<fast_to>,<slow_int>` | Advertisement timing                             |
| `SW,<pin>,<func>`                     | Pin function configuration                       |

### SR -- Feature Bitmap

| Bit    | Feature                         |
| ------ | ------------------------------- |
| 0x8000 | Hardware flow control (CTS/RTS) |
| 0x4000 | No Prompt                       |
| 0x2000 | Fast Mode (no $$$)              |
| 0x0100 | UART Transparent without ACK    |
| 0x0080 | Reboot after Disconnection      |
| 0x0040 | Running Script after Power On   |
| 0x0020 | MLDP streaming service          |
| 0x0010 | Disable DLE (v1.28+)            |
| 0x0008 | Command Mode Guard (v1.28+)     |

### SS -- Services Bitmap

| Bit  | Service            |
| ---- | ------------------ |
| 0x80 | Device Information |
| 0x40 | UART Transparent   |
| 0x20 | Beacon             |

**Warning:** Issuing `SS` removes all custom-defined services. Use before `PS`/`PC`.

### Action Commands

| Command           | Description                  |
| ----------------- | ---------------------------- |
| `$$$`             | Enter Command mode           |
| `---\r`           | Exit to Data mode            |
| `A[,<int>,<to>]`  | Start advertisement          |
| `Y`               | Stop advertisement           |
| `F[,<int>,<win>]` | Start scan                   |
| `X`               | Stop scan                    |
| `C,<0/1>,<addr>`  | Connect (0=public, 1=random) |
| `K,1`             | Disconnect                   |
| `B`               | Bond current connection      |
| `R,1`             | Reboot                       |
| `D`               | Dump device info             |
| `V`               | Firmware version             |
| `U,<1-8/Z>`       | Remove bonding               |

### Service Configuration

| Command                   | Description                               |
| ------------------------- | ----------------------------------------- |
| `PS,<UUID>`               | Define service UUID                       |
| `PC,<UUID>,<prop>,<size>` | Add characteristic                        |
| `PZ`                      | Clear all private services (needs reboot) |

**Characteristic Properties:**

| Bit  | Property               |
| ---- | ---------------------- |
| 0x02 | Read                   |
| 0x04 | Write without response |
| 0x08 | Write                  |
| 0x10 | Notify                 |
| 0x20 | Indicate               |

Common: `0x02`=Read, `0x0A`=Read+Write, `0x1A`=Read+Write+Notify

### Characteristic Access

| Command               | Description                                  |
| --------------------- | -------------------------------------------- |
| `SHR,<handle>`        | Read local characteristic                    |
| `SHW,<handle>,<data>` | Write local characteristic (triggers notify) |
| `CHR,<handle>`        | Read remote characteristic                   |
| `CHW,<handle>,<data>` | Write remote characteristic                  |
| `CI`                  | Init client role (service discovery)         |

## Initialization Sequence (Peripheral with Transparent UART)

```
$$$                      -> CMD>
SF,1\r                   -> (reboots, %REBOOT%)
$$$                      -> CMD>
SN,MeteoStation\r        -> AOK
SS,C0\r                  -> AOK   // Device Info + Transparent UART
SA,2\r                   -> AOK   // No Input No Output
SR,4000\r                -> AOK   // No prompt (for MCU parsing)
R,1\r                    -> %REBOOT%
```

## Reference Circuit (RN4871)

```
MCU TX   --> RN4871 UART_RX (pin 7)
MCU RX   <-- RN4871 UART_TX (pin 8)
MCU GPIO --> RN4871 RST_N   (pin 10)
3.3V     --> RN4871 VBAT    (pin 14)
GND      --> RN4871 GND     (pins 2, 13)
```

Recommended: 4.7k pull-up + 1 uF cap on RST_N, 10 uF bulk cap on VBAT.

If using flow control (SR,8000):

```
MCU CTS <-- RN4871 P3_6/RTS (pin 9)
MCU RTS --> RN4871 P0_0/CTS (pin 11)
```
