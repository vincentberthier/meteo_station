# BME280 -- Combined Digital Humidity, Pressure and Temperature Sensor

**Manufacturer:** Bosch Sensortec
**Document:** BST-BME280-DS001-24, Rev 1.24, February 2024
**Package:** 2.5 mm x 2.5 mm x 0.93 mm metal-lid LGA, 8 pins

## Device Overview

The BME280 is a combined digital humidity, pressure, and temperature sensor. It is register- and performance-compatible with the BMP280 digital pressure sensor. Each sensing element can be independently enabled or disabled. Supports both I2C (up to 3.4 MHz) and SPI (up to 10 MHz).

Three power modes:

- **Sleep mode** (default after POR): no measurements, lowest power (0.1 uA typ).
- **Forced mode**: one measurement cycle, then returns to sleep.
- **Normal mode**: perpetual cycling between measurement and standby.

## I2C Details

### Addresses

- **SDO = GND**: `0x76`
- **SDO = VDDIO**: `0x77`

Supports Standard, Fast, and High Speed modes. No clock stretching. Recommended pull-ups: 4.7 kOhm.

### Protocol

- **Write:** address pairs NOT auto-incremented. Send (register, data) pairs.
- **Read:** set register address in write mode, then repeated-start in read mode. Auto-increments. **Burst reads strongly recommended** for data consistency.

## Compensation Algorithms (32-bit Integer)

**Temperature** (returns 0.01 C, e.g. "5123" = 51.23 C):

```c

BME280_S32_t BME280_compensate_T_int32(BME280_S32_t adc_T)
{
    BME280_S32_t var1, var2, T;
    var1 = ((((adc_T>>3) - ((BME280_S32_t)dig_T1<<1))) * ((BME280_S32_t)dig_T2)) >> 11;
    var2 = (((((adc_T>>4) - ((BME280_S32_t)dig_T1)) * ((adc_T>>4) - ((BME280_S32_t)dig_T1)))
        >> 12) * ((BME280_S32_t)dig_T3)) >> 14;
    t_fine = var1 + var2;
    T = (t_fine * 5 + 128) >> 8;
    return T;
}
```

**Pressure** (returns Pa as Q24.8, e.g. "24674867" = 96386.2 Pa):

## Initialization Sequence (I2C, Forced Mode)

1. Wait 2 ms after power-on
2. Read chip ID (0xD0), verify = 0x60
3. Soft reset (optional): write 0xB6 to 0xE0, wait 2 ms
4. Wait for NVM copy: poll 0xF3 bit 0 until 0
5. Read calibration: 0x88-0xA1 (26 bytes) and 0xE1-0xE7 (7 bytes)
6. Write ctrl_hum (0xF2): humidity oversampling
7. Write ctrl_meas (0xF4): temp/pressure oversampling + mode=forced (**must come after ctrl_hum**)
8. Wait for measurement: poll 0xF3 bit 3 or wait t_measure_max
9. Burst read 0xF7-0xFE (8 bytes)
10. Compensate: temperature first (for t_fine), then pressure, then humidity

**Ordering rule:** Always write ctrl_hum (0xF2) **before** ctrl_meas (0xF4).
