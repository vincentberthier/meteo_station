Project Overview Embedded Rust Weather Station

MeteoStation is a bare-metal weather station firmware written in Rust for the STM32H753ZI microcontroller. It uses the Embassy async runtime to read environmental sensors and stream data over Bluetooth Low Energy. A companion CLI tool on the host can scan, connect, and display live readings.

What It Measures

🌡

Temperature

BMP388 sensor, -40 to +85 °C range, 0.005 °C resolution. Reads once per second.

☁

Barometric Pressure

BMP388 sensor, 300–1250 hPa range, 0.016 Pa resolution at highest oversampling.

📡

BLE Streaming

RN4871 module broadcasts temperature and pressure as GATT characteristics with notify support.

Architecture Embassy Async

The firmware runs on the Embassy async executor with no heap allocator. Each peripheral gets its own task. Tasks communicate through bounded channels, never through shared mutable state. The library crate is hardware-agnostic and tested on the host.

Library Design

meteo-lib depends only on embedded-hal-async traits, never on concrete hardware. This means all drivers can be unit-tested on the host with mock implementations. The BMP388 driver, RN4871 protocol parser, encoding helpers, and GATT handle discovery all have test suites that run via cargo nextest on x86_64.

sensors::bmp388

Generic over I2c trait. Reads calibration from NVM, applies Bosch compensation algorithm.

ble::driver

Generic over Uart trait. Manages command/response flow with the RN4871.

ble::line_buffer

Fixed-size circular buffer for UART line framing. No allocations. Handles \r\n and %...% delimited events.

ble::encoding

Little-endian f32 encode/decode, hex conversion helpers for BLE wire format.

Hardware STM32H753ZI Nucleo-144

The station runs on a Nucleo-H753ZI board (Cortex-M7 @ 480 MHz, 2 MB flash, 1 MB RAM). Sensors connect over I2C and UART. All I/O is 3.3 V — the STM32H7 is not 5 V tolerant.

BLE Protocol RN4871 + Custom GATT

The firmware communicates with the RN4871 BLE module over UART using ASCII commands. It defines a custom GATT service with two characteristics for temperature and pressure, both supporting read and notify operations.

GATT Service

UUID Type Properties Size
a4e64b8b-...-7d3c3f2e1a00 Service — —
a4e64b8b-...-7d3c3f2e1a01 Temperature Read Notify 4 bytes (f32 LE)
a4e64b8b-...-7d3c3f2e1a02 Pressure Read Notify 4 bytes (f32 LE)

BLE Task Startup Sequence

Phase 1

Hardware Reset

Assert RST_N for 100 ms, then wait for %REBOOT% status event.

  • Pull PA4 low (100 ms)
  • Release RST_N
  • Scan UART for %REBOOT%
Phase 2

Configuration

Enter command mode, configure device, define GATT services.

  • Send $$$, wait for CMD>
  • Set name to "MeteoStation"
  • Clear private services (PZ)
  • Define service + 2 characteristics
  • Reboot to activate
Phase 3

Streaming

Monitor connections and write sensor data to GATT handles.

  • Discover handles via LS
  • Start advertising
  • On connect + reading: SHW
  • Adaptive timeout (100 ms / 5 s)

RN4871 Command Protocol

The driver wraps RN4871 ASCII commands in a typed Command enum. Each variant knows its wire format and expected response type (AOK, single line, multi-line, or reboot).

Command Wire Format Response Purpose
SetName SN,<name>\r AOK Set BLE device name
DefineService PS,<32hex>\r AOK Create GATT service
DefineCharacteristic PC,<uuid>,<props>,<size>\r AOK Add characteristic
ServerWrite SHW,<handle>,<hex data>\r AOK Write to characteristic
ListServices LS\r Multi-line Discover handles

Status Events

The module sends asynchronous %...%-delimited events over UART. The LineBuffer extracts these, and the status parser categorizes them:

%REBOOT%

Module finished booting. Safe to enter command mode.

%CONNECT,...%

Client connected. Contains address type and MAC address.

%DISCONNECT%

Client disconnected. Stop writing characteristics.

%WC,handle,data%

Client wrote to a handle (e.g. CCCD enable for notifications).

Host CLI Client

meteo-cli uses btleplug to scan for a peripheral named "MeteoStation", connect, discover services, and subscribe to notification streams. It prints temperature (°C) and pressure (Pa / hPa) as readings arrive.

Logging

Firmware uses defmt for structured logging over RTT (Real-Time Transfer) through the ST-LINK JTAG connection. Log level is controlled by the DEFMT_LOG environment variable (default: trace). No UART or serial output is used for logs.

Testing Strategy

All logic in meteo-lib is tested on the host. Drivers are generic over embedded-hal-async traits, so tests use mock implementations (e.g. MockUart for the RN4871 driver). Tests run via cargo nextest run --lib --target x86_64-unknown-linux-gnu.

Hardware-interfacing code in meteo-firmware cannot be automatically tested. Manual validation uses just run with RTT log inspection.

probe-rs Safety

Never kill probe-rs run with SIGTERM or a timeout. This leaves the debug probe locked and the chip halted, requiring a physical unplug. Always send SIGINT (Ctrl+C) for clean detachment.
